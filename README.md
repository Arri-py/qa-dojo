# QA Dojo

> *Додзё — место, где рождается мастерство.*

Здесь ты будешь учиться работать с Git, Docker и тестированием.  
Каждое задание — это **ката**: упражнение, которое нужно выполнить и понять.

Не гугли сразу. Сначала попробуй сам.

## Как приступить к кате

```bash
# Получить актуальное состояние репозитория
git fetch

# Переключиться на нужную ветку
git checkout kata/01-calculator
```

Открой `README.md` в ветке — там полное описание задания.

## Правила додзё

1. **Первое правило додзё — читай логи. Второе правило додзё — ЧИТАЙ ЛОГИ.**
2. Скопировал код и не понял как он работает — считай, не написал.
3. Если что-то сломалось, сначала попробуй починить сам. Минимум 15 минут. Потом гугли.
4. `it works on my machine` — не ответ. Докер существует именно для того, чтобы это не было ответом.
5. Коммить часто. Коммит — это сохранение в игре. Не сохранился — сам виноват.
6. Если что-то работает, но ты не знаешь почему — это не работает.
7. Stack Overflow — твой сенсей. Но сенсей тоже ошибается: проверяй дату ответа.
