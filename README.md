### Шаг 2 — Изучить API

Открой документацию в браузере:

```
http://localhost:8088/docs
```

Swagger UI покажет все эндпоинты — сгруппированные по версиям **v1** и **v2**.  
Попробуй **каждый** эндпоинт вручную прямо в браузере, прежде чем писать код.

#### Справочник эндпоинтов

Калькулятор хранит **историю вычислений** — каждый результат получает уникальный `id`.

| Метод | Эндпоинт | Что делает |
|-------|----------|------------|
| `GET` | `/api/v1/health` | Проверить, жив ли сервис |
| `GET` | `/api/v1/operations` | Список доступных операций |
| `POST` | `/api/v1/calculations` | Создать вычисление |
| `GET` | `/api/v1/calculations` | История всех вычислений |
| `GET` | `/api/v1/calculations/{id}` | Получить вычисление по ID |
| `PUT` | `/api/v1/calculations/{id}` | Заменить вычисление (все поля) |
| `PATCH` | `/api/v1/calculations/{id}` | Изменить вычисление частично |
| `DELETE` | `/api/v1/calculations/{id}` | Удалить вычисление |

**v2** расширяет v1: добавляет операции `modulo` и `power`, в ответах появляется поле `expression` (`"10.0 × 5.0 = 50.0"`), и новый эндпоинт `GET /api/v2/calculations/{id}/expression`.

| | v1 | v2 |
|---|----|----|
| Операции | `add subtract multiply divide` | + `modulo power` |
| Поле `expression` в ответе | нет | есть |

### Шаг 3 — Подготовить рабочее место (venv)

Прежде чем писать код, создай изолированное окружение для своего проекта.  
Это стандартная практика — зависимости клиента не должны конфликтовать с системным Python.

```bash
# создать окружение в папке .venv
python3 -m venv .venv

# активировать (Linux / macOS)
source .venv/bin/activate

# активировать (Windows)
.venv\Scripts\activate
```

После активации приглашение в терминале изменится — появится `(.venv)`:

```
(.venv) $
```

Установи зависимость и зафиксируй её:

```bash
pip install requests
pip freeze > requirements.txt
```

Когда закончишь работу — деактивируй окружение:

```bash
deactivate
```

> Папку `.venv` не коммитят в git — она уже добавлена в `.gitignore`.

### Шаг 4 — Написать консольный клиент

Напиши Python-скрипт `client.py`. Он должен пройти через все типы запросов и обе версии API в следующем порядке:

#### Блок A — работа с v1

1. `GET /api/v1/health` — убедиться что сервис жив, вывести статус
2. `GET /api/v1/operations` — получить и вывести список операций v1
3. `POST /api/v1/calculations` — создать вычисление (например, `add 10 + 5`), сохранить `id`
4. `GET /api/v1/calculations` — вывести всю историю
5. `GET /api/v1/calculations/{id}` — получить созданное вычисление по ID
6. `PATCH /api/v1/calculations/{id}` — изменить одно число (например, `b` → `99`), вывести новый результат
7. `PUT /api/v1/calculations/{id}` — полностью заменить вычисление (другая операция и числа), вывести результат
8. `DELETE /api/v1/calculations/{id}` — удалить вычисление
9. `GET /api/v1/calculations/{id}` — убедиться что запись удалена (ожидаем 404)

#### Блок B — работа с v2

10. `GET /api/v2/operations` — получить и вывести список операций v2 (сравни с v1)
11. `POST /api/v2/calculations` — создать вычисление с операцией `power` (например, `2 ^ 10`), обрати внимание на поле `expression` в ответе
12. `GET /api/v2/calculations/{id}/expression` — получить результат в виде строки
13. `POST /api/v2/calculations` — создать вычисление с операцией `modulo`
14. `PATCH /api/v2/calculations/{id}` — изменить операцию и число, вывести обновлённый `expression`
15. `DELETE /api/v2/calculations/{id}` — удалить оба вычисления v2

#### Блок C — граничные случаи

16. `POST /api/v1/calculations` с делением на ноль — обработать ошибку 400 и вывести сообщение
17. `GET /api/v1/calculations/несуществующий-id` — обработать ошибку 404

Пример вывода для одного шага:

```
[POST /api/v1/calculations]
  Запрос:  {"operation": "add", "a": 10, "b": 5}
  Ответ:   {"id": "3fa85f64-...", "operation": "add", "a": 10.0, "b": 5.0, "result": 15.0}

[PATCH /api/v1/calculations/3fa85f64-...]
  Запрос:  {"b": 99}
  Ответ:   {"id": "3fa85f64-...", "operation": "add", "a": 10.0, "b": 99.0, "result": 109.0}
```

> Для HTTP-запросов используй библиотеку `requests`.
