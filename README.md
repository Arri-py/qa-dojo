# Ката #02 — Тест-сан

---

## Легенда

Калькулятор из первой каты снова в деле.  
На этот раз твоя задача — не пользоваться им, а **проверить его**.  
Настоящий мастер не доверяет инструменту на слово.

---

## Что дано

```
app/                      — калькулятор (тот же что в кате #01)
docker-compose.yml        — запуск калькулятора
tests/
├── conftest.py           — базовая фикстура (изучи, она подскажет паттерн)
└── test_calculator.py    — сюда пишешь тесты (TODO внутри)
```

---

## Задание

### Шаг 1 — Поднять калькулятор

```bash
docker-compose up --build -d
```

Убедись что сервис жив:

```
http://localhost:8088/docs
```

### Шаг 2 — Настроить окружение

```bash
python3 -m venv .venv
source .venv/bin/activate

pip install pytest requests
pip freeze > requirements.txt
```

### Шаг 3 — Изучи conftest.py

Открой `tests/conftest.py` — там две фикстуры.  
Разберись как они работают прежде чем писать тесты.  
`created_calculation` показывает важный паттерн: **setup → yield → teardown**.

### Шаг 4 — Написать тесты

Открой `tests/test_calculator.py` — внутри TODO с описанием каждого теста.  
Реализуй их по блокам:

#### Блок A — базовые проверки
Убедись что сервис отвечает и возвращает ожидаемые данные.

#### Блок B — параметризация
Один тест на все операции v1, один на операции только из v2.  
Используй `@pytest.mark.parametrize`.

#### Блок C — CRUD-сценарий
Полный цикл: создать → прочитать → обновить → удалить.  
Используй фикстуру `created_calculation` из conftest.

#### Блок D — v2 и поле expression
Проверь что v2 возвращает поле `expression` и что отдельный эндпоинт работает.

#### Блок E — граничные случаи
Деление на ноль и несуществующий id — оба должны возвращать правильные статус-коды.

### Шаг 5 — Запустить тесты

```bash
pytest tests/ -v
```

Все тесты должны пройти:

```
tests/test_calculator.py::test_health PASSED
tests/test_calculator.py::test_operations PASSED
...
```
